## 1. 오늘의 목표
단순한 표 계산을 넘어, 수만 개의 데이터를 유기적으로 연결하는 **Pandas**의 설계 철학을 이해합니다. **DataFrame**과 **Series**가 왜 데이터 분석의 표준이 되었는지 그 내부 구조를 분석합니다.

---

## 2. 데이터의 틀: 구조적 분석

### 1) 판다스: `Pandas`
* **어원:** **Pan**el **Da**ta **S**ystem (패널 데이터 시스템).
* **심층 분석:** 여러 대상을 장기간 추적하는 '패널 조사'에서 유래했습니다. 즉, 판다스는 단순히 숫자를 나열하는 게 아니라 **시간과 항목이라는 두 축을 가진 입체적 데이터**를 추적하기 위해 설계된 도구입니다.

### 2) 데이터프레임: `DataFrame`
* **구조:** **Data** + **Frame** (뼈대/틀).
* **심층 분석:** 모든 열(Column)이 동일한 인덱스(Index)라는 **공통의 뼈대**를 공유합니다. 이 '프레임' 덕분에 서로 다른 성격의 데이터(이름, 나이, 점수)가 뒤섞이지 않고 정교한 표 형태로 유지됩니다.

---

## 3. 실무적 핵심: 데이터의 신상 파악

### 1) 정보 요약: `df.info()`
* **분석:** 데이터의 타입과 **결측치(Null)** 유무를 즉시 파악합니다. 머신러닝 모델에 넣기 전 '데이터의 건강검진' 단계입니다.

### 2) 기술 통계: `df.describe()`
* **분석:** 수치형 데이터의 분포(평균, 사분위수)를 확인하여 **이상치(Outlier)**가 있는지 판단하는 기준이 됩니다.

---

## 4. 실습 코드 (심층 분석 주석)

```python
import pandas as pd

# 1. 데이터프레임 생성: 서로 다른 자료형의 결합
data = {
    'Item': ['A', 'B', 'C'],
    'Price': [100, 200, 150],
    'Stock': [True, False, True]
}
df = pd.DataFrame(data)

# 2. Series 분석: DataFrame에서 한 줄을 떼어내면 'Series'가 됨
# 시리즈는 고유의 인덱스를 유지하는 '연속된 사슬'입니다.
price_series = df['Price']

# 3. 데이터 요약 확인
print(df.describe())
```

## 🧭 오늘의 핵심 분석 및 요약
1. **Pandas** = 입체적인 흐름을 추적하는 패널 데이터 전문 도구
2. **DataFrame** = 서로 다른 데이터(Series)들을 'Index'라는 공통의 뼈대로 묶어낸 구조체
3. **Columnar Storage** = 열 단위 처리를 통해 대용량 데이터 계산 효율을 극대화함
4. **Data Integrity** = info()와 describe()를 통해 데이터의 무결성을 먼저 확인하는 것이 분석의 본질
