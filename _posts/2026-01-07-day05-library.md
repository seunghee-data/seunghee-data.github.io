## 1. 오늘의 목표
파이썬의 진정한 힘은 '공유'에 있습니다. 전 세계 천재들이 설계한 **모듈(Module)**과 **라이브러리(Library)**가 어떻게 내 코드에 이식되는지 그 메커니즘을 어원으로 분석합니다.

---

## 2. 외부 지식의 이식: Import

### 1) 수입하다: `import`
* **어원:** **Im**(내부로) + **Port**(항구). 
* **심층 분석:** 항구에서 컨테이너를 들여오듯, 외부 파일에 저장된 함수와 클래스 뭉치를 현재 내 파일의 메모리 공간으로 **실어 나르는(Transport)** 행위입니다. 단순히 가져오는 게 아니라 사용할 수 있는 '경로'를 연결하는 것입니다.

### 2) 별칭: `as`
* **어원:** **Alias**(가명)의 줄임표현.
* **심층 분석:** 복잡한 외부 이름을 내 코드 환경에 맞게 **재정의**하는 것입니다. 협업 시 가독성을 높이기 위해 `pandas`를 `pd`로 부르는 것은 일종의 '업계 약속(Convention)'입니다.

---

## 3. 구조적 단위: Module vs Library

### 1) 모듈: `module`
* **어원:** **Modulus**(작은 단위). 
* **심층 분석:** 레고 블록 하나가 '모듈'입니다. 특정 기능(예: 수학 계산, 랜덤 생성)을 수행하는 가장 작은 **파일 단위(.py)**를 뜻합니다.

### 2) 라이브러리: `library`
* **어원:** **Liber**(책) + **-ary**(장소).
* **심층 분석:** 수많은 모듈(책)들이 모여 있는 **도서관**입니다. 우리는 이 도서관에서 원하는 기능을 대출해 쓰기만 하면 됩니다. 파이썬이 '배터리 포함(Batteries Included)' 언어라 불리는 이유는 이 도서관이 매우 방대하기 때문입니다.

---

## 4. 실습 코드 (심층 분석)

```python
# 1. 특정 모듈에서 필요한 책만 빌려오기
from math import sqrt, pi   # math 도서관에서 '제곱근'과 '파이'만 콕 집어 가져옴

# 2. 별칭으로 사용 편의성 높이기
import numpy as np          # 대규모 수치 계산 모듈을 'np'라는 가명으로 이식

# 3. 라이브러리 활용의 실제
result = sqrt(16)           # math 모듈의 함수 사용
print(f"16의 제곱근: {result}")
print(f"원주율: {pi}")
```

## 🧭 오늘의 핵심 요약
1. **Import** = 항구(Port)를 통해 외부 지식을 내 메모리로 수입하기
2. **As** = 코드의 효율성을 위해 가명(Alias)을 부여하는 관례
3. **Module** = 특정 기능을 담은 독립적인 코드 부품 (파일 단위)
4. **Library** = 수많은 모듈이 체계적으로 정리된 거대한 지식 창고
