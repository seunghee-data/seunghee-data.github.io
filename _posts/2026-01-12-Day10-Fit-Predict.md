## 1. 오늘의 목표
10일간의 여정의 종착역입니다. 파이썬으로 데이터를 다루는 궁극적인 이유인 **머신러닝(Machine Learning)**의 내부 메커니즘을 분석합니다. 기계가 학습한다는 것이 수학적으로 어떤 의미인지 파헤칩니다.

---

## 2. 학습과 예측의 메커니즘

### 1) 학습시키다: `fit`
* **어원:** **Fit** (옷을 몸에 맞추다 / 꼭 맞다).
* **심층 분석:** 모델이 데이터라는 '몸'에 자신의 논리라는 '옷'을 딱 맞게 수선하는 과정입니다. 데이터의 패턴(기울기와 절편 등)을 찾아내어 오차를 최소화하는 상태로 모델을 고정하는 핵심 단계입니다.

### 2) 예측하다: `predict`
* **어원:** **Pre**(미리) + **Dict**(말하다).
* **심층 분석:** `fit`을 통해 완성된 논리 체계에 새로운 데이터를 투입하여, 그 결과가 무엇일지 미리 연산하여 내뱉는 과정입니다.

---

## 3. 구조적 분석: 왜 학습과 검증을 나누는가?
* **과적합(Overfitting) 방지:** 훈련 데이터에만 너무 'fit'하게 맞추면, 실제 새로운 데이터에서는 엉터리 결과를 냅니다. 그래서 우리는 데이터를 **Train(공부용)**과 **Test(시험용)**로 엄격히 분리하여 모델의 진짜 실력을 검증합니다.

---

## 4. 실습 코드 (머신러닝 워크플로우 분석)

```python
from sklearn.linear_model import LinearRegression

# 1. 모델 생성 (기계의 뼈대 준비)
model = LinearRegression()

# 2. 학습 (Fit)
# 기계가 데이터의 패턴에 자신의 수식을 맞춥니다.
model.fit(X_train, y_train)

# 3. 예측 (Predict)
# 공부한 내용을 바탕으로 미지의 데이터 결과를 출력합니다.
predictions = model.predict(X_test)

# 4. 평가 (Evaluation)
# 실제 정답과 예측값을 비교해 모델의 '정확도'를 측정합니다.
```

## 🧭 오늘의 핵심 분석 및 요약
1. **Fit** = 모델의 내부 파라미터를 데이터 패턴에 최적화하여 '몸에 맞추는' 과정
2. **Feature & Label** = 예측의 재료(독립변수)와 정답(종속변수) 사이의 상관관계를 찾는 것
3. **Generalization** = 머신러닝의 진정한 목표는 연습 데이터가 아닌 '처음 본 데이터'를 잘 맞추는 것
4. **Machine Learning** = 명시적인 프로그래밍 없이 데이터 스스로가 규칙을 찾아내게 만드는 통계적 기술
